## Iteration 4 — 2026-04-15T06:30Z
**Focus**: Silent logic error — calibration endpoint uses wrong probability
**Finding**: `/api/calibration` used `entry_price` (market mid price at trade time) as the forecast probability for Brier score computation. This measures market calibration, not model calibration. The model's actual probability was never stored in paper trade records.
  - Before: `forecasts.append(price)` where price = market mid
  - After: `forecasts.append(model_prob)` where model_prob = model's computed probability
**Change**:
  - src/verification/paper_trader.py: Added `model_probability` parameter to `record_trade()`
  - src/orchestrator/pipeline.py: Pass `model_probability=model_prob` to paper_trader
  - src/api/main.py: Use `model_probability` for Brier score, fallback to entry_price for legacy trades

## Iteration 8 — 2026-04-15T07:50Z
**Focus**: Runtime bug — BUY_NO trades executed at YES token price
**Finding**: Two compounding bugs in pipeline.py Pass 2 (lines 249-251):
  1. `snap` variable referenced in Pass 2 was stale — it held the last station from Pass 1's loop, not the station for the current pending item. If the last Pass 1 station was Miami but the pending item was NYC, the NO token lookup hit Miami's price dict.
  2. Even with the correct snapshot, the fallback `snap.market_prices.get(no_token_id, price)` used the YES token's MarketPrice when the NO token was missing. This caused buying NO tokens at the YES ask price ($0.62 instead of ~$0.42) — overpaying by ~$0.20 per dollar.
  - Also added foregone conclusion guard in EdgeDetector: skip when market_prob > 0.92 or < 0.08 (near-certain outcomes).
**Change**:
  - src/orchestrator/pipeline.py: Look up NO token price during Pass 1 (correct `snap`), store in `pending["no_price"]`. Pass 2 uses stored price, not stale `snap`.
  - src/trading/edge_detector.py: Added `max_market_certainty` param (default 0.92). SKIP when market near-certain.
  - src/config/settings.py: Added `MAX_MARKET_CERTAINTY` setting.
  - main.py: Wire new setting to EdgeDetector.
  - tests/unit/test_orchestrator.py: Added `test_buy_no_uses_no_token_price_not_yes` integration test.
  - tests/unit/test_trading.py: 6 new tests for foregone conclusion guard.
